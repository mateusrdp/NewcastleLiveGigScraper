# Project structure

- ./docs/ is where all documentation is
  - I keep documentation simple and mostly to guide AI code generating agents
- ./calendars/*ics is where all ics calendars are saved
  - ./calendars/recurring_events.ics is produced manually and must not be deleted
- ./src/filters is where all the filters are stored (one venue infix per line, comparisons are case insensitive)
  - calendars generated by filters or merges are prefixed with "gigs_". Eg: "./calendars/gigs_mat_locals.ics"
- ./src/ is where all source code is
  - main.py is the entry point
  - merge_ics.py postprocesses and merges all scripts generated by the scrapers
  - generate_debug_index.py and generate_index.py generates the github pages
  - scraper_debug.py is a helper to save debug info
- ./src/scrapers/*.py is where all the individual scrapers are
  - every scraper in this dir is a single file script named <site>_scraper.py
  - scraper scripts take no parameters
  - every scraper in this dir creates a corresponding calendar with the same name in the "../calendars/" dir, ommitting the "_scraper.py" suffix and adding the ".ics" extension. Eg: ./src/scrapers/newcastlelive_scraper.py -> ./calendars/newcastlelive.ics
- ./github/workflows/*.yml is where all the GitHub actions are
